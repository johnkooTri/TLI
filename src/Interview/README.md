# getter,setter

인스턴스 변수는 어째서 private으로 선언하는것이 정석인가?
getter,setter 를 이용해서 접근할 수 있게 하면 차라리 public으로 선언해버리는것이 나은게 아닌가?


객체가 인스턴스 변수를 private 로 선언하는데에는 캡슐화 라는 객체지향 프로그래밍 원리를 구현하기 위해서
우리 말로는 **정보은닉**

객체는 레퍼런스를 이용해서 접근하게 되는데 여로곳에서 레퍼런스를 가지고 있게 되면 정보의변경이 문제가 될수 있다. 따라서 **객체의 데이터를 누구나 손댈 수 없게 할 필요가 있어서 private 접근제한자 사용**

데이터성 객체의 경우에는 데이터를 은닉하면 사용하기 어려워지니 조회와 세팅을 할수있게 getter,setter 사용

  **getter는 데이터의 복사본을 던져주기 때문에 원래 객체의 데이터를 손상시키지 않습니다.**
  **간단하게 생성자를 통해서만 데이터를 넣어줄수 있고 (객체 주입) ,getter 메소드만 있다면 외부에서는 객체의 원본 데이터를 변경할 수 없게 된다. 이런경우 불변(Immutable)이라고 한다.**

 - **setter 메소드를 이용하면 파라미터를 검증할 수 있다.**

 - **setter,getter는 객체의 데이터의 수정에 대해 무방비 상태가 되어 객체지향의 객체의 데이터를 보호하자는 원칙에   위배된다.**
